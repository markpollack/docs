## Workflow Builder

```java
Workflow.<I, O>define("name")          // → WorkflowBuilder
    .step(step)                         // add first step
    .then(step)                         // chain next step
    .branch(predicate)                  // conditional routing
        .then(step).otherwise(step)
    .parallel(step1, step2)            // concurrent execution
    .repeatUntilOutput(predicate)      // loop (do-while)
        .step(step).end()
    .decision(chatClient)              // LLM routing
        .option("name", step).end()
    .gate(gate)                        // quality checkpoint
        .onPass(step).onFail(step)
        .maxRetries(3).end()
    .onError(ExType.class, recovery)   // error recovery (on last step)
    .run(input)                        // execute immediately
    .run(input, RunOptions.maxCost(1.0)) // with constraints
    .build()                           // → reusable Workflow (implements Step)
    .compile()                         // → WorkflowGraph (IR only)
```

## Step Interface

```java
@FunctionalInterface
public interface Step<I, O> {
    O execute(AgentContext ctx, I input);

    default String name() { ... }       // for traces
    default Class<?> inputType() { ... }
    default Class<?> outputType() { ... }
}
```

**Factory methods:**

```java
Step.named("name", (ctx, input) -> output)  // lambda with explicit name
Step.noop()                                  // identity pass-through
```

## Built-in Step Types

### ChatClientStep (single LLM call)

```java
// String → String
Step<String, String> step = ChatClientStep.of(chatClient, "Summarize: {input}");

// String → typed response
Step<String, MyResponse> step = ChatClientStep.of(
    chatClient, "Classify: {input}", MyResponse.class);
```

### ClaudeStep (full agent session)

```java
Step<String, String> agent = ClaudeStep.of("Fix the tests in {input}")
    .workingDirectory(projectPath)
    .permissionMode(PermissionMode.ACCEPT_EDITS)
    .withMcp(mcpConfig);
```

### Deterministic Steps

```java
Steps.of((ctx, input) -> transform(input))  // with context
Steps.of(input -> transform(input))          // without context
Steps.terminate(WorkflowStatus.FAILED, "reason")
Steps.outputOf("prior-step-name")            // retrieve earlier output
Steps.retrying(3, riskyStep)                 // retry on any exception
Steps.retrying(3, TimeoutException.class, step)  // selective retry
```

## Gate Interface

```java
@FunctionalInterface
public interface Gate<O> {
    GateDecision evaluate(AgentContext ctx, O output);
}
```

**`GateDecision`**: `PASS`, `FAIL`, `ESCALATE`, `TIMEOUT`

### Built-in Gates

**JudgeGate** — backed by Agent Judge jury:

```java
Gate<Object> gate = new JudgeGate(jury, 0.7);  // score >= 0.7 → PASS
```

**TieredGate** — three outcomes:

```java
Gate<Object> gate = new TieredGate(jury, 0.8, 0.4);
// score >= 0.8 → PASS
// score >= 0.4 → ESCALATE
// score < 0.4  → FAIL
```

### Gate Builder Options

```java
.gate(qualityGate)
    .onPass(approveStep)           // required
    .onFail(rejectStep)            // optional
    .onTimeout(fallbackStep)       // optional
    .withReflector(reflectorStep)  // transforms verdict → feedback text
    .maxRetries(3)                 // retry cap on FAIL
    .end()
```

## Supervisor Builder

```java
Workflow.supervisor("name", routingChatClient)
    .agents(reviewStep, editStep, formatStep)
    .until(ctx -> ctx.get(AgentContext.ITERATION_COUNT).orElse(0) >= 5)
    .maxIterations(10)             // fallback safety cap
    .run(input)
    .run(input, RunOptions.maxCost(2.0))
    .build()                       // → reusable Workflow
```

## AgentContext

Immutable, threaded through all steps. Copy-on-write via `mutate()`.

### Well-known Keys

| Key | Type | Description |
|-----|------|-------------|
| `WORKFLOW_RUN_ID` | `String` | Unique execution ID |
| `WORKFLOW_NAME` | `String` | Workflow name |
| `CURRENT_STEP` | `String` | Active step name |
| `ITERATION_COUNT` | `Integer` | Loop iteration counter |
| `ACCUMULATED_COST` | `Double` | USD spend so far |
| `ACCUMULATED_TOKENS` | `Long` | Token consumption |
| `JUDGE_VERDICT` | `Object` | Gate failure verdict |
| `JUDGE_REFLECTION` | `String` | Reflector-generated feedback |

### Usage

```java
// Read
Optional<Integer> count = ctx.get(AgentContext.ITERATION_COUNT);
int required = ctx.require(AgentContext.ITERATION_COUNT); // throws if missing

// Write (returns new context)
AgentContext next = ctx.mutate()
    .with(AgentContext.ACCUMULATED_COST, 1.50)
    .build();
```

## RunOptions

```java
RunOptions.maxCost(2.0)              // stop if cost exceeds $2
RunOptions.maxIterations(50)         // stop after 50 steps
RunOptions.maxDuration(Duration.ofMinutes(5))
RunOptions.unlimited()               // no constraints

// Chain
RunOptions opts = RunOptions.maxCost(2.0)
    .withMaxIterations(50)
    .withMaxDuration(Duration.ofMinutes(5));
```

## WorkflowGraph (IR)

The DSL compiles to an immutable intermediate representation:

```java
WorkflowGraph<I, O> graph = Workflow.define("name")
    .step(a).then(b)
    .compile();

graph.name()                    // workflow name
graph.nodes()                   // List<WorkflowNode>
graph.edges()                   // List<WorkflowEdge>
graph.startNode()               // entry node name
graph.finishNode()              // exit node name
graph.nodeByName("step-name")   // lookup
```

### Node Types (sealed)

| Node | Description |
|------|-------------|
| `StepNode` | Regular step execution |
| `GatewayNode` | Predicate-based branch |
| `DecisionNode` | LLM-driven routing |
| `GateNode` | Quality gate with retry |
| `LoopEntryNode` | While-do loop start |
| `LoopCheckNode` | Do-while condition check |
| `LoopExitNode` | Loop exit point |
| `ForkNode` | Parallel split |
| `JoinNode` | Parallel / branch convergence |

### Edge Conditions (sealed)

| Condition | From |
|-----------|------|
| `Unconditional` | Sequential edges |
| `BooleanGuard(true/false)` | GatewayNode |
| `OptionMatch("name")` | DecisionNode |
| `GateMatch(PASS/FAIL)` | GateNode |
| `BranchIndex(i)` | ForkNode |
| `ErrorMatch(ExType)` | Error edges |
| `LoopContinue` / `LoopExit` | Loop nodes |

## AgentLoopAdvisor

Drop-in Spring AI advisor for autonomous tool-calling agents:

```java
AgentLoopAdvisor.builder()
    .toolCallingManager(manager)      // Required
    .maxTurns(20)                      // Default: 20
    .timeout(Duration.ofMinutes(10))   // Default: 10 min
    .costLimit(5.0)                    // Default: $5.00
    .stuckThreshold(3)                 // Default: 3 same outputs
    .jury(myJury, 5)                   // Optional: evaluate every N turns
    .listener(myListener)              // Optional: event notifications
    .build();
```

### AgentLoopListener

```java
public interface AgentLoopListener {
    default void onLoopStarted(String runId, String userMessage) {}
    default void onTurnStarted(String runId, int turn) {}
    default void onTurnCompleted(String runId, int turn,
            TerminationReason reason) {}
    default void onLoopCompleted(String runId, LoopState finalState,
            TerminationReason reason) {}
    default void onLoopFailed(String runId, LoopState state,
            Throwable error) {}
}
```

### TerminationReason

| Value | Description |
|-------|-------------|
| `COMPLETED` | No more tool calls |
| `MAX_TURNS_REACHED` | Hit `maxTurns` |
| `TIMEOUT` | Exceeded `timeout` |
| `COST_LIMIT_EXCEEDED` | Exceeded `costLimit` |
| `STUCK_DETECTED` | Same output `stuckThreshold` times |
| `JURY_PASSED` | Jury evaluation passed |
| `ABORTED` | External abort signal |
