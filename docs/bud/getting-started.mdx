## Prerequisites

- **Java 21+** on PATH
- **Claude Code** installed and authenticated ([claude.ai/code](https://claude.ai/code))
- **JetBrains IDE** 2026.1+ with AI Assistant plugin enabled

## Install from Release

Download both JARs from the [v0.2.0 release](https://github.com/markpollack/bud/releases/tag/v0.2.0) to a directory (e.g. `~/bud/`):

- `bud-acp-server-0.2.0-jar-with-dependencies.jar` — the ACP agent
- `bud-mcp-server-0.2.0.jar` — the MCP tool server

## Install from Source

```bash
git clone https://github.com/markpollack/bud.git
cd bud
./mvnw package -DskipTests
```

This produces:
- `bud-acp-server/target/bud-acp-server-*-jar-with-dependencies.jar`
- `bud-mcp-server/target/bud-mcp-server-*.jar`

## Configure JetBrains

Create or edit `~/.jetbrains/acp.json`:

```json
{
  "agent_servers": {
    "Bud": {
      "command": "java",
      "args": [
        "-Dbud.mcp.jar=/path/to/bud-mcp-server-0.2.0.jar",
        "-jar",
        "/path/to/bud-acp-server-0.2.0-jar-with-dependencies.jar"
      ]
    }
  }
}
```

<Warning>
Replace `/path/to` with the absolute path where you saved the JARs. The `-Dbud.mcp.jar` system property tells the ACP agent where to find the MCP tool server.
</Warning>

Restart your JetBrains IDE after editing the config.

## Your First Project

1. Open any project in IntelliJ
2. Open **AI Chat** tool window
3. Select **"Bud"** from the agent dropdown
4. Type: `create a REST app called hello-api in /tmp`

Bud will scaffold a complete Spring Boot REST project with:
- `GreetingController` with GET endpoint
- `GreetingService` with business logic
- `ProblemDetail` error handling
- Working tests
- Maven wrapper

## Verify

```bash
cd /tmp/hello-api
./mvnw test
```

## How It Works

When you send a message to Bud in AI Chat:

1. JetBrains sends the prompt to Bud's ACP agent over stdio
2. Bud's LLM classifier (Sonnet) analyzes the intent — capabilities, domain entities, template match
3. The execution planner routes to the right tier:
   - **CACHE** — matches a built-in template, instant scaffolding
   - **INITIALIZR** — generates via start.spring.io (Boot 3.x, Gradle)
   - **EXPAND** — scaffolds a base then generates domain-specific code
4. Tools execute deterministically — no LLM in the actual scaffolding
5. Results stream back through ACP to your chat window

The first prompt may take 30-60 seconds (CLI startup + classification + MCP server startup). Subsequent prompts in the same session are faster.
