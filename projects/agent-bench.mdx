Spring AI Bench measures agents on real enterprise development tasks — issue triage, PR review, coverage uplift, compliance validation, dependency upgrades. Run benchmarks on your repos to measure your scenarios. Agent-agnostic: Claude, Gemini, Amazon Q, Amp, or any CLI tool that reads a file and writes files.

**If agents have evolved, benchmarks must evolve too.** SWE-bench measures 2023 Python patches. Spring AI Bench measures 2025 enterprise Java workflows.

## How It Works

<Steps>
  <Step title="Provide">
    `bench provide` copies a workspace template and writes `INSTRUCTION.md` with the task.
  </Step>
  <Step title="Run any agent">
    Claude, Gemini, a bash script — whatever reads the instruction and modifies the workspace.
  </Step>
  <Step title="Grade">
    `bench grade` evaluates the workspace using YAML-configured judges from [Agent Judge](/projects/agent-judge).
  </Step>
</Steps>

The filesystem is the contract. The provide/grade separation means any CLI tool can participate — no SDK integration required.

## Benchmark Tracks

| Track | Status | What it measures |
|-------|--------|-----------------|
| **Hello World** | Ready | File creation and basic infrastructure |
| **Code Coverage** | In development | Coverage uplift on Spring projects |
| **Issue Analysis** | In development | Automated triage and classification |
| **PR Review** | In development | Risk assessment and structured reports |
| **Static Analysis** | In development | Fix code quality issues, preserve functionality |

## CLI

```bash
bench list                                          # Discover benchmarks
bench provide --benchmark X --item Y --workspace DIR  # Set up workspace
bench grade --benchmark X --item Y --workspace DIR    # Evaluate results
bench run --benchmark X --agent claude-code           # Full pipeline
```
