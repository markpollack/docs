title: "Agent Judge"
description: "Agent-agnostic evaluation framework with deterministic, command, and LLM judges"
---

Agent Judge is a pluggable evaluation framework for verifying AI agent task completion. It provides three kinds of judges ŌĆö deterministic rules, command execution, and LLM-powered assessment ŌĆö that compose into juries with configurable voting strategies. Zero coupling to any specific agent implementation.

The core module (`agent-judge-core`) has **zero external dependencies**. Specialized modules add command execution (`agent-judge-exec`) and LLM evaluation (`agent-judge-llm`).

## Core Abstractions

<CardGroup cols={2}>
  <Card title="Judge" icon="gavel">
    Functional interface ŌĆö takes `JudgmentContext`, returns `Judgment` with score, status, reasoning, and granular checks
  </Card>
  <Card title="Jury" icon="users">
    Multi-judge aggregation with voting strategies ŌĆö majority, consensus, weighted average, median
  </Card>
</CardGroup>

## Judge Types

| Type | Module | Cost | Example |
|------|--------|------|---------|
| **Deterministic** | `agent-judge-core` | Free | `FileExistsJudge`, `FileContentJudge`, custom rules |
| **Command** | `agent-judge-exec` | Compute only | `BuildSuccessJudge` (Maven/Gradle), `CommandJudge` |
| **LLM** | `agent-judge-llm` | Token cost | `CorrectnessJudge`, custom LLM evaluation |
| **Agent** | `agent-judge-agent` | Agent cost | Delegate evaluation to an AI agent |
