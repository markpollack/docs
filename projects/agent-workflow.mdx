## Overview

Agent Workflow is a Java library for building multi-step agentic pipelines. You define **steps** — each one calls an LLM, runs a function, or invokes an external agent — and compose them into **workflows** using a fluent DSL with branching, loops, quality gates, and parallel execution. Steps exchange data through a typed **context**, and the workflow compiles to a **graph IR** that separates definition from execution. This means the same workflow code runs on different runtimes without changes — local today, with crash recovery or durable execution as the project matures.

```java
Workflow.define("pr-review")
    .step(fetchDiff)
    .then(analyzeDiff)
    .gate(new JudgeGate(jury, 0.8))
        .onPass(postComment)
        .onFail(revise)
    .end()
    .run(event);
```

## Core Concepts

**Steps** are the building blocks. Each step takes input, does work, and produces output. Steps can be:

- **Deterministic** — a Java function (GitHub API call, string formatting, file parsing)
- **LLM-powered** — a `ChatClientStep` (single Spring AI call), a `ClaudeStep` (full multi-turn agent session), or an `AgentClientStep` (external agent runtime)

A `ClaudeStep` isn't a single API call — it runs a complete agentic loop internally (dozens of tool calls, minutes of execution) and returns a typed result. The workflow sees it as one step.

**Context** threads through every step. Steps read input parameters by key, do their work, and write output parameters back. Downstream steps pick up what upstream steps produced — all type-safe via `ContextKey<T>`.

**The graph IR** means the workflow definition is pure data — nodes and edges, not opaque lambdas. This enables:
- **Portable runtimes** — the IR decouples definition from execution. Additional runtimes (crash recovery via `workflow-batch`, durable execution) planned without changing workflow code
- **Tracing** — every step transition is recorded for observability and [Markov analysis](/methodology/markov-fingerprinting)
- **Inspection** — the graph is pure data (nodes + edges), not opaque lambdas
