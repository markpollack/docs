## Overview

Agent Workflow is a Spring-native DSL for composing multi-step agentic pipelines. Developers write fluent workflow definitions that compile to a `WorkflowGraph` IR, execute via a swappable `PartitionHandler`, and record every transition for tracing and behavioral analysis.

```java
Workflow.define("pr-review")
    .step(fetchDiff)
    .then(analyzeDiff)
    .gate(new JudgeGate(jury, 0.8))
        .onPass(postComment)
        .onFail(revise)
    .end()
    .run(event);
```

## Why It Matters

**Step = full agentic loop.** A `ClaudeStep` runs a multi-turn LLM session internally ŌĆö dozens of tool calls, minutes of execution ŌĆö and returns a typed result. The workflow orchestrates *semantic* steps; the loop mechanics are internal. This is the opposite of frameworks where each LLM API call is a separate workflow activity.

**Durability is a deployment concern.** The same workflow code runs on `LocalPartitionHandler` (zero overhead), `CheckpointingPartitionHandler` (JDBC crash recovery), or `TemporalPartitionHandler` (distributed durability). Single `@Bean` swap ŌĆö no workflow code changes.

**10+ composable primitives.** Sequential, branch, while-do loop, do-while loop, parallel, dynamic parallel, LLM decision routing, quality gates with verdict feedback, supervisor, error recovery, terminate. The graph IR uses Java 21 sealed interfaces ŌĆö every construct produces real nodes and typed edges, not opaque lambdas.
