## Overview

Agent Workflow lets you compose **steps** into a **workflow**. Each step does one thing ã call an LLM, run a deterministic function, invoke an external agent. Steps pass data to each other through a typed **context** (key-value pairs that flow through the pipeline). The workflow definition compiles to an **intermediate representation** (a graph of nodes and edges), which means the same workflow code can execute in different ways ã in-process, with JDBC crash recovery, or on Temporal ã without changing a line.

```java
Workflow.define("pr-review")
    .step(fetchDiff)
    .then(analyzeDiff)
    .gate(new JudgeGate(jury, 0.8))
        .onPass(postComment)
        .onFail(revise)
    .end()
    .run(event);
```

## Core Concepts

**Steps** are the building blocks. Each step takes input, does work, and produces output. Steps can be:

- **Deterministic** ã a Java function (GitHub API call, string formatting, file parsing)
- **LLM-powered** ã a `ChatClientStep` (single Spring AI call), a `ClaudeStep` (full multi-turn agent session), or an `AgentClientStep` (external agent runtime)

A `ClaudeStep` isn't a single API call ã it runs a complete agentic loop internally (dozens of tool calls, minutes of execution) and returns a typed result. The workflow sees it as one step.

**Context** threads through every step. Steps read input parameters by key, do their work, and write output parameters back. Downstream steps pick up what upstream steps produced ã all type-safe via `ContextKey<T>`.

**The graph IR** means the workflow definition is pure data ã nodes and edges, not opaque lambdas. This enables:
- **Crash recovery** ã if the app restarts mid-workflow, completed steps are skipped and execution resumes from the last checkpoint
- **Tracing** ã every step transition is recorded for observability and [Markov analysis](/methodology/markov-fingerprinting)
- **Visualization** ã the graph is inspectable at compile time

**Portable runtimes.** Same workflow code, different durability ã swap with a single `@Bean`:

| Runtime | What it adds | Requires |
|---------|-------------|----------|
| `LocalPartitionHandler` | Nothing ã in-process, zero overhead | Nothing (default) |
| `CheckpointingPartitionHandler` | JDBC crash recovery | A `DataSource` |
