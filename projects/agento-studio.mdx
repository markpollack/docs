Agento Studio is a systematic approach to growing AI agents. A **judge** tells you if the agent got it right. A **journal** tells you why it behaved the way it did. You need both — without the judge you're guessing, without the journal you're tuning blind.

The toolkit gives you the feedback loop: skills, knowledge bases, judges, journals, and the eval harness to prove the whole stack actually improves outcomes. Built on the **Forge methodology** — a six-phase system that separates discovery from execution and treats evaluation as first-class.

## How It Works

1. **Run** the agent on a real task
2. **Judge** the output — did it actually work? (correctness)
3. **Read the journal** — why did it behave that way? (behavior)
4. **Fix the hotspot** — better skill, better prompt, better tool
5. **Run again** — measure if it improved

Each lever you can turn — skills, knowledge bases, pre-analysis, steering hooks — gets validated through this loop, not assumed to help.

```bash

```

## What's Inside

### Methodology

Every project starts with structured discovery before any code is written:

- **Vision** — what are we building and why
- **Design** — how will it work, what are the trade-offs
- **Roadmap** — step-by-step execution plan with QA review

4 project variants (eval-agent, project, research, steward) share the same six-phase Forge methodology with different feedback loops. 13 concept docs and 11 templates make it repeatable.

### Knowledge Bases

Agent-native knowledge systems using routing tables instead of RAG. Two types: **Research-Partner** (broad, theme-based synthesis) and **Code-Agent** (narrow, task-driven lookup with YAML frontmatter). Federation connects KBs across projects. Health checks keep them honest.

### Agents

Software development agents built and grown through the judge + journal loop:

| Agent | What it does |
|-------|-------------|
| [PR Merge](/projects/loopy) | Automated pull request review, merge, and pipeline orchestration |
| [Issue Classification](/experiments/issue-classification) | Categorize and triage issues (tested against SWE-bench) |
| [Code Coverage](/experiments/code-coverage-v3) | Generate tests to hit coverage targets across multiple projects |
| Liquibase → Flyway | Migration agent for database schema tooling conversion |

More agents are in development. Each one goes through the same cycle: run on real tasks, judge the output, read the journal, fix the hotspots, run again.

| `/forge-steward` | Bootstrap ongoing project stewardship |
