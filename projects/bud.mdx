## Overview

Bud is a Spring Boot development agent that understands what you're building, modifies existing projects through conversation, and generates domain code following Spring Boot conventions when patterns aren't enough. Not another template-driven scaffolder — an LLM classifies intent and expands domain code when templates can't cover it. It runs as an ACP agent in JetBrains, Zed, VS Code — any ACP-compatible editor.

**Zero API keys.** Bud delegates to your enterprise's approved CLI for all LLM calls. The agent carries no model credentials — your IT department controls the AI provider.

## Key Features

- **23 MCP tools** — `create_project`, `add_dependency`, `add_actuator`, `add_security`, `analyze_project`, and more
- **9 project templates** — REST, JPA, Security, Actuator, Batch, Scheduling, Thymeleaf, Spring AI, Minimal
- **Enterprise-safe** — All tools are deterministic (no LLM calls). The LLM comes from your approved CLI.
- **IDE-native** — Shows up in JetBrains AI Chat dropdown via ACP protocol
- **Production templates** — Constructor injection, slice tests, ProblemDetail errors, proper layering

## Architecture

```
IDE (JetBrains / Zed / VS Code)
  └─ ACP Client
       └─ Bud ACP Agent
            └─ AgentClient → Claude Code CLI
                 └─ Bud MCP Server (23 tools)
                      └─ bud-core (pure Java library)
```
